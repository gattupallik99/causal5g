Threshold-based alerting

Every NF that crosses a KPI threshold fires simultaneously. In a cascade, that's every downstream NF of the faulted one — 6 alarms for a single root cause. Alert storms obscure signal.

Correlation-based grouping

AIOps platforms group co-occurring alarms by time proximity. They can tell you "AMF and PCF alarmed together." They cannot tell you which caused the other — or that mIoT is fine.

The key gap: slice-topology awareness

5G SA defines Network Slices as logical end-to-end networks with different NF compositions. A PCF fault in a URLLC slice does not affect an mIoT slice that has no PCF. No existing RCA system is aware of this topology. Causal5G's Level-2 slice sub-DAG is the first published implementation of slice-topology-aware causal attribution.

The key gap: feedback recalibration

Existing 5G fault management systems use static rule tables or fixed ML models. Causal5G's recalibrator updates causal edge weights based on whether remediation actually fixed the fault — a genuine closed feedback loop. After 5 successful PCF remediations, the PCF→SMF causal edge is permanently stronger in the DAG. The system improves with each incident.
