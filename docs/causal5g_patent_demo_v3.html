Threshold-based alerting

Every NF that crosses a KPI threshold fires simultaneously. In a cascade, that's every downstream NF — 6 alarms for a single root cause. Alert storms obscure signal.

Correlation-based grouping

AIOps platforms group co-occurring alarms by time proximity. They cannot tell you which caused the other — or that mIoT is completely fine.

How It Works
Five-stage causal inference pipeline

Granger causalityA statistical test that determines if one time series causally precedes another — not just correlates. If PCF metrics degraded 8s before AMF, Granger establishes PCF → AMF direction. + PC algorithmPeter-Clark algorithm: a constraint-based causal discovery method that builds a directed acyclic graph (DAG) from conditional independence tests between NF metric time series. + Bayesian inference, fused across four telemetry streams, producing a bi-level causal DAGTwo-layer directed acyclic graph: Level-1 maps NF-to-NF causal edges; Level-2 maps the fault to specific network slices. Both layers in a single FaultReport. every 30 seconds.
